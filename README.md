# EvoLab 係咩？

用一句講：**教電腦自己試錯，學做決定。**

唔係人手寫「見到紅燈就停」呢啲死規則，而係：

1. 隨機出一班「做法」
2. 睇邊個做得好
3. 好嘅留低生仔
4. 仔有少少突變（改少少）
5. 重複幾百輪

主線做法叫 **遺傳演算法**（英文 Genetic Algorithm）——自然界用呢招進化動植物；呢度就用嚟進化「玩遊戲／扮炒股票」嘅方法。  
股票仲有另一頁叫 **蒙地卡羅**（網址 `/stock-mc`）：考題一樣，不過搵答案方法唔同——**亂試一大堆做法，留最好嗰個，再試多輪**。

首頁 **演算法原理**（`/theory`）用圖畫 + 小實驗，解釋呢套「生仔進化」點運作、同呢個 project 點實踐。

> **重要：** 呢個專案係 **教學示範**，唔係投資軟件。股票頁只係示範演算法，**唔係投資建議**。  
> **唔好當提款機。**

## 核心概念（生活化版）

### 1. 族群（一班學生）

每一代有幾十個「學生」（例如 36 個）。  
每個學生代表 **一套做法**。  
專業名：**族群（population）**。

### 2. 基因體（學生嘅筆記）

每個學生身上有一串數字，叫 **基因體**（英文 genome）。  
可以想像成考試溫書筆記：

- 貪食蛇／打磚塊／Flappy Bird：筆記 = 大腦入面「見到咩就點做」嘅參數
- 股票：筆記 = **指標點樣調** + **點決定買賣**

### 3. 適應度（考試分數）

每個學生考一次試，攞一個 **分數**（專業名 fitness）：

- 蛇：食到幾多、行得幾耐
- 打磚塊：清幾多磚、清得夠快唔夠
- Flappy Bird：過幾多條水管、飛得幾耐
- 股票：有冇贏過「一路買住唔郁手」；亂買賣、成世坐現金會扣分

**分數高 → 較容易生仔傳去下一輪。**

### 4. 一代之內發生咩事？

好似一班人考試完再收生：

```
一班學生考試
    ↓
分數高嘅幾個「保送」直接入下一代（精英）
    ↓
其餘多數位：抽兩個家長 → 合併筆記 → 加少少突變 → 成新學生
    ↓
最後一個位塞一個「外來插班生」（保持新鮮諗法）
    ↓
成新一班，再考
```

#### 抽家長（錦標賽）

每次隨機抽 3 個學生出嚟比併，分數最高嗰個做阿爸／阿媽。  
差生都有少少機會（撞啱同更差嘅同組）——唔好殺晒「試新嘢」。  
好處：就算全班分數好接近，「邊個叻啲」照樣分得出。

專業名：**錦標賽選擇（tournament selection）**。

#### 合併筆記（雜交）

仔女每一格筆記，一半機會抄阿爸、一半機會抄阿媽。  
好似兩本筆記各撕一半砌埋。

專業名：**雜交（crossover）**。

#### 突變

合併完之後，有機會改亂幾格：

- 多數（約 80%）：喺原數上面郁少少
- 少數（約 20%）：整格重寫

用意：避免成班人永遠困喺同一個壞習慣。

專業名：**突變（mutation）**。

#### 精英

最好嗰幾個 **原封不動** 帶落下一代。  
免得因為一時運氣差，無啦啦冇咗今代冠軍。

專業名：**精英保留（elitism）**。

#### 插班生

每代最後一個位，換一個新面孔。  
股票頁特別啲：插班生 **只重抽「指標點樣調」**，大腦決策部分可以沿用——持續試新週期／門檻，又唔使次次打亂已經學識嘅買賣習慣。

專業名：**移民（immigrant）**。

### ① 貪食蛇（`/snake`）

**要學咩：** 唔撞牆／自己、盡量食果。

#### 進化設定（預設）

| 項目     | 數值 | 白話                    |
| -------- | ---- | ----------------------- |
| 族群大小 | 36   | 每代 36 套蛇策略        |
| 突變率   | 0.12 | 約 12% 機會改某一格筆記 |
| 突變幅度 | 0.24 | 改動時郁幾大下          |
| 精英比例 | 0.08 | 約保留 2–3 個直接落代   |
| 播放速度 | 5    | 預設全速                |

#### 大腦結構

```
10 個輸入 → 12 個中間神經元 → 3 個輸出
```

| 項目       | 內容（白話）                                                   |
| ---------- | -------------------------------------------------------------- |
| 基因數量   | **171** 個數字（全部係大腦入面嘅參數）                         |
| 輸入（10） | 前／左／右有冇危險、食物喺邊（左右上下距離）、而家面朝邊、身長 |
| 輸出（3）  | 左轉 / 直行 / 右轉（揀分最高嗰個）                             |

#### 分數權重（fitness）

每個策略用 **兩個固定運氣編號**（137、911）各打一場，再 **取平均**（減少單場好運）。

一場分數：

| 項目         | 權重／公式                            | 白話                   |
| ------------ | ------------------------------------- | ---------------------- |
| 食果（主要） | `食到數量² × 180`                     | 食得愈多，獎勵飆得愈勁 |
| 食果（額外） | `食到數量 × 90`                       | 再加一層鼓勵           |
| 存活         | `行咗幾步 × 0.28`                     | 唔好一早撞死           |
| 接近食物     | 靠近 **+0.16**；遠離 **−0.2**（每步） | 早期教「要行近果」     |

```
一場分數 ≈ 食到²×180 + 食到×90 + 步數×0.28 + 靠近／遠離加減分
總分 = (編號137一場 + 編號911一場) ÷ 2
```

其他規則：格 **20×20**；太耐唔食會餓死（約 `110 + 已食×16` 步）；最多 **10,000** 步。

### ② 打磚塊（`/breaker`）

**要學咩：** 板跟波、接住、清磚。

#### 進化設定（預設）

| 項目     | 數值 | 白話                 |
| -------- | ---- | -------------------- |
| 族群大小 | 12   | 物理重，所以學生少啲 |
| 突變率   | 0.14 | 比蛇高少少           |
| 突變幅度 | 0.26 | 改動大少少           |
| 精英比例 | 0.10 | 約保留 1 個精英      |
| 播放速度 | 5    | 預設全速             |

#### 大腦結構

```
8 個輸入 → 12 個中間神經元 → 3 個輸出
```

| 項目      | 內容（白話）                                         |
| --------- | ---------------------------------------------------- |
| 基因數量  | **147** 個數字                                       |
| 輸入（8） | 板左右位置、球位置／速度、最近磚相對位置、仲剩幾多磚 |
| 輸出（3） | 向左 / 停 / 向右                                     |

#### 分數權重（fitness）

每個策略用 **五場真·隨機場景**（發球角度、板位、球位、磚格微移、撞板／撞磚抖動）各打一場，再 **取平均**——逼電腦真係識跟波，唔好背死一條路線。  
畫面循環重播 **每圈再抽一場**，你會睇到唔同版本。

一場分數：

| 項目       | 權重                                      | 白話                             |
| ---------- | ----------------------------------------- | -------------------------------- |
| 清磚       | 每磚 **× 140**                            | 主力：清得多先係王道             |
| 全清獎勵   | 清晒全部磚 **+2000**                      | 大獎                             |
| 全清速度   | `(剩時間比例)² × 2200 + 剩時間比例 × 500` | **愈快清晒分差愈大**             |
| 消磚效率   | `(清磚數 / 步數) × 1400`                  | 未全清都獎「消得密」             |
| 跟波加分   | 波喺下半場對準板（有上限）                | 早期學跟波；唔可以靠呢樣蓋過清磚 |
| 多餘接波罰 | 超出「清磚×2+4」每次 **−6**               | **禁止**淨碌住接波、唔消磚刷分   |

```
一場分數 ≈ 清磚×140
         +（全清 ? 2000 + 速度獎 : 0）
         + 消磚效率
         + 跟波（有上限）
         − 多餘接波×6
總分 = 幾場隨機場景分數平均
```

例：同樣全清，**300 步搞掂** 分數遠遠高過 **3000 步先搞掂**（速度獎可以差過千）。  
唔獎「接波次數／拖時間」——否則電腦會淨係碌波唔消磚。

物理：場約 **560×420**；板闊 **92**；球速大約夾喺 **4–7**；最多 **3,600** 步。

### ④ 股票交易（`/stock`）

**要學咩：** 點調技術指標參數、幾時買／持／賣。

#### 進化設定（預設）

| 項目     | 數值   | 白話                                    |
| -------- | ------ | --------------------------------------- |
| 族群大小 | 150    | 每代 150 套策略（筆記多，要夠人一齊試） |
| 突變率   | 0.12   | 基數；頭／尾再乘倍數（見下）            |
| 突變幅度 | 0.22   | 基數；頭／尾再乘倍數                    |
| 精英比例 | 0.08   | 約保留 2–3 個精英                       |
| 播放速度 | 5      | 預設全速（代同代之間唔刻意等人）        |
| 神經網絡 | **關** | 預設用簡單規則投票（較易明）；可自己開  |

#### 基因體分兩邊（好重要）

總共約 **281** 個數字 = **18 個頭** + **263 個大腦參數**。

| 區段 | 數量 | 內容（白話）         | 突變（相對基數）                  |
| ---- | ---- | -------------------- | --------------------------------- |
| 頭   | 18   | 指標週期、買賣門檻等 | 改得 **密 3 倍**、幅度 **1.5 倍** |
| 尾   | 263  | 決策大腦參數         | 改得 **少啲**（約 0.55 倍）       |

白話：**多用時間試「指標點樣調」**；大腦都改，但改得輕過參數。  
插班生：只重抽頭（參數），保留尾。  
咁樣可以戒甩「日日買完又賣」嘅壞習慣，又唔使每次打亂成個大腦。

#### 大腦結構（神經網絡開時）

```
18 個輸入 → 10 個中間 → 5 個中間 → 3 個輸出（買／持／賣）
```

18 樣大致：快慢均線、威廉、ROC、RSI、MACD、保力加、波動、成交量、**N 日新高**、**N 日新低**、而家有冇持倉、同買賣門檻差幾遠……  
**唔餵** 開／高／低／收（K 線結構）。全部指標常開；「有冇持倉」永遠開。

#### 網絡點樣落倉（神經網絡開時）

輸出係 **買／持／賣** 三個分數，唔係亂揀最大就郁：

| 規則                              | 白話                             |
| --------------------------------- | -------------------------------- |
| 買要 **贏「持」至少 0.08** 先開倉 | 分數貼身時唔好日日跳             |
| 賣要 **贏「持」至少 0.08** 先平倉 | 同上                             |
| 已持倉時，「買」當留守唔當加倉    | 買同賣差唔多時，唔好亂翻         |
| 日日亂買賣用 **分數罰**（換手罰） | **冇**「最少持幾日先准賣」嘅硬鎖 |

關神經網絡時，用下面「規則模式」+ 同一套換手罰，一樣 **唔硬鎖** 持倉日數。

#### 固定用嘅指標

移動平均線、威廉、ROC、RSI、MACD、保力加通道、波動率、成交量、**N 日新高**、**N 日新低**（NN 輸入唔再包開高低收）。

| 指標     | 計法（白話）                                    | 大腦睇到咩     |
| -------- | ----------------------------------------------- | -------------- |
| N 日新高 | 過去 N 日最高價；收市愈近呢個高，愈貼「創新高」 | 突破／追高訊號 |
| N 日新低 | 過去 N 日最低價；收市愈近呢個低，愈貼「創新低」 | 支撐／貼低訊號 |

兩個「望幾耐」**各自一個數字**（約 10–120 日），進化可以分開調；圖表下拉可揀「N 日高」／「N 日低」睇線。

#### 成交假設（同複製出去嘅腳本對齊）

- **今日收市先決定 → 第二日開市先成交**（唔係即買即得）
- 每次買賣單邊成本約 **0.15%**（手續費／滑價咁理解）
- **冇**「最少持幾日先准賣」硬鎖；日日亂翻靠 **成本 + 分數換手罰** 打壓

#### 分數權重（fitness）

- 時間切兩段：**60% 訓練入分**；**尾 40% 純測試永不入收生**
- 起始虛擬本金 **10,000**

```
總分 ≈ 訓練×0.85 + 穩健×0.15
        − 大腦參數唔好太大（開神經網絡時）
```

**一段分數——贏大市先係主力；淨跟大市 ≈ 0 分：**

主軸係「你最終有幾多錢 ÷ 一路買住唔賣最終有幾多錢」——  
同大市打和 = 0 分，**贏大市先有主力分**。  
另外獎實際賺到、肯入市；罰跌得深、日日翻倉、成世坐現金。

| 項目           | 權重（大概）     | 白話                                        |
| -------------- | ---------------- | ------------------------------------------- |
| 每年贏大市幾多 | × **220**        | 主力                                        |
| 成段合共贏幾多 | × **45**         | 輔力                                        |
| 實際賺到幾多   | × **55**         | 唔好淨「打和超額」                          |
| 平均有冇入市   | × **24**         | 牛市肯買；少坐現金                          |
| 路徑穩唔穩     | × **5**          | 少少分                                      |
| 最大回吐       | × **−12**        | 跌得深扣分                                  |
| 平均換手幾密   | **−48t − 140t²** | 罰日日買完又賣；愈密罰愈重（二次方）        |
| 成世坐現金     | **−40 / −12**    | 入市比例好低：約低於 8% 重罰、低於 20% 輕罰 |

**半段穩健：** 訓練再切兩半，  
`0.35 × 較差半 + 0.65 × 兩半平均`——唔好淨前半靚後半仆。

開局會塞幾個「經典菜式」做法（第一個接近「死買唔賣」：易買、難賣），  
唔使由完全亂數字慢慢爬。

> **期望管理：** 日線只做多 + 0.15% 成本 + 次日開盤，**穩定打贏 QQQ 本身好難**。  
> 揀冠軍時睇 **測試回報**，唔好淨睇訓練回報。Demo 成功 ≠ 實盤印鈔。

#### 規則模式（關神經網絡時）

唔用 263 個大腦參數；改為：

- 均線／MACD／RSI／威廉 **多數贊成**（至少 2 票）→ 買
- **賣出有趨勢閘：**
    - 上升趨勢（快線 > 慢線）：要 **RSI 同威廉一齊過熱** 先賣
    - 否則：RSI **或** 威廉過熱 → 賣

#### 畫面額外顯示

- 訓練／測試回報、同「一路買住唔賣」對照
- 冠軍指標參數（含 N 日新高／新低週期）
- 圖表可切：**均線+保力加**／動量／MACD／風險／**N 日高**／**N 日低**
- 側欄 **訓練控制** 可以：
    - **複製 Pine Script**（貼去 TradingView）
    - **複製富途 Python**（貼去富途牛牛自編指標）

### ⑤ 蒙地卡羅（`/stock-mc`）——用「亂試」搵炒法

先用一句講晒：

> **股票交易頁** 同 **蒙地卡羅頁**，考嘅係 **同一張卷**（幾時買、幾時賣、指標點調）。  
> 分別只係：**點樣搵好答案** 唔同。

| 頁面                    | 好似做咩                                                                       |
| ----------------------- | ------------------------------------------------------------------------------ |
| 股票交易（`/stock`）    | 好學生生仔：叻嘅人結婚、仔女抄筆記再亂改少少                                   |
| 蒙地卡羅（`/stock-mc`） | 買六合彩式亂試：一次抽好多張「做法紙」，計分，**最好嗰張永遠留低**，下一輪再抽 |

「蒙地卡羅」個名聽落好大路——其實就係 **靠大量亂試去估答案** 嘅老方法（賭場、天氣預報都有人用）。  
呢度唔係教你賭錢，只係借呢種 **「試好多、留最好」** 嘅精神。

#### 想像：煲湯試味

你想煲一煲好湯，唔知鹽落幾多、火候點先啱：

1. **一次試 40 煲**（每批樣本數，預設 40）
2. 其中大約 **三分一**，係抄而家最好嗰煲，再 **加減少少鹽**（喺冠軍附近再磨）
3. 其餘 **三分二**，完全亂嚟：鹽多鹽少、有人落薑有人唔落
4. 每煲都「試味」打分（分數計法同股票交易頁 **一模一樣**）
5. 如果有煲贏過舊冠軍 → **換佢做新冠軍**；舊冠軍唔會因為一次運氣差就俾人掟走
6. 再煲下一批 40 煲……重複幾百輪

第一批仲會塞幾煲「經典菜式」（例如常見均線做法），唔好一開始完全亂嚟。

## 股票頁：點解有「訓練」同「測試」？

想像你溫書（而家程式切兩段）：

- **前約 60% 日子** = 溫習卷（**訓練**）  
  電腦用呢段時間計分數、揀邊個學生生仔／留冠軍。
- **最後約 40% 日子** = 真正生卷（**測試**）  
  **完全唔入收生**。只係俾你睇：溫習好嘅人，考生卷係咪一樣叻。

如果訓練好靚、測試又仆，好多時係 **背熟舊卷、唔係真本事**（專業名：過度擬合）。  
呢個正正係股票 demo 想教嘅一課。  
**股票交易** 同 **蒙地卡羅** 兩頁用 **同一套溫習卷／生卷切法、同一套計分**，所以可以公平比較：邊種搵答案方法叻啲。

## 常見問題

### 點解股票訓練好慢？

因為每一套做法，都要喺好多年股價上面「假裝交易」一次。  
一次試愈多人／愈多套、歷史愈長，愈慢。

想快：

- 播放速度拉去 **5**
- 「族群大小」或「每批樣本數」細少少
- 試下關神經網絡（改用簡單規則）

### 點解訓練回報靚、測試又仆？

好正常。  
等於背熟溫習卷，生卷唔識。  
記住：**測試段唔入訓練**，就係要揭穿呢種假叻。

### 「生仔」同「蒙地卡羅」邊個好？

**冇一定——呢個就係想你自己試。**

- **生仔（股票交易頁）：** 攞兩個幾好嘅做法砌埋，有時穩、有時困死
- **蒙地卡羅：** 唔砌人，就係 **亂試 + 抄冠軍再改**；好易明，有時撞到寶，有時飄忽

同一張分卷，開兩個分頁一齊跑，睇測試回報同參數設定，自己判斷。

### 搵到「必勝參數」未？

**未，亦唔應該咁諗。**  
呢啲頁示範嘅係「電腦點樣自己試錯搵做法」，**唔係保證未來賺錢**。

### 複製 Pine Script／富途 Python 係咩？

冠軍出嚟之後，喺右側 **訓練控制** 可以（內容複製到剪貼簿，唔再下載檔）：

| 掣                   | 產出                 | 用喺邊                   |
| -------------------- | -------------------- | ------------------------ |
| **複製 Pine Script** | Pine 全文 → 剪貼簿   | TradingView 圖表策略     |
| **複製富途 Python**  | Python 全文 → 剪貼簿 | 富途牛牛自編 Python 指標 |

兩邊都由同一份冠軍筆記生成（參數 + 大腦／規則）。  
富途版會記住「有倉先平、空倉先開」，同埋而家有冇持倉餵返入大腦；畫均線／保力加／**N 日高／N 日低** + 買賣標記。  
一樣：**只供學習，唔係實盤建議。**  
股票交易頁同蒙地卡羅頁共用；冠軍檔案兩邊可以互用。  
（如果舊檔案係好舊版本嘅大腦結構，可能匯入唔到，要重新訓練一次。）

### 點解股票「train 唔出必勝模型」？

好正常，原因包括：

1. **日線只做多 + 0.15% 成本 + 次日開盤**，穩定打贏 QQQ「一路買住」本身好難
2. 分數主軸係 **贏大市幾多**——跟大市 ≈ 0 分，唔係「賺到錢就滿分」
3. 大腦參數好多，進化又冇「倒數教」（backprop），容易卡喺日日亂翻或者成世坐現金
4. 訓練靚、測試仆 = 背熟舊卷（過度擬合）

而家已加：**訓練前後半穩健分**、**預設規則模式**、**換手 thrash 罰**。  
揀冠軍時對齊 **測試回報** 同「一路買住」；淨訓練靚唔夠。
