# EvoLab 係咩？

用一句講：**教電腦自己試錯，學做決定。**

唔係人手寫「見到紅燈就停」咁啲死規則，而係：

1. 隨機出一班「策略」
2. 睇邊個做得好
3. 好嘅留下來生仔
4. 仔有少少突變
5. 重複幾百代

主線做法叫 **遺傳演算法（Genetic Algorithm）**——自然界進化物種；呢度進化「玩遊戲／炒股票」嘅方法。  
股票仲有另一頁叫 **蒙地卡羅**（`/stock-mc`）：考題一樣，不過搵答案嘅方法唔同——**亂試一大堆做法，留最好嗰個，再試多輪**。

> **重要：** 呢個專案係 **教學 demo**，唔係投資軟件。股票頁只係示範演算法，**唔係投資建議**。

## 核心概念（生活化版）

### 1. 族群（一班學生）

每一代有幾十個「學生」（例如 36 個）。  
每個學生代表 **一套做法**。

### 2. 基因體（學生嘅筆記）

每個學生身上有一串數字，叫 **基因體（genome）**。  
可以想像成考試溫書筆記：

- 貪食蛇／打磚塊：筆記 = 大腦入面「見到咩就點做」嘅參數
- 股票：筆記 = **指標點調** + **點決定買賣**

### 3. 適應度（考試分數）

每個學生考一次試，攞一個 **分數（fitness）**：

- 蛇：食到幾多、行得幾耐
- 打磚塊：清幾多磚、接住幾耐
- 股票：訓練期跑贏「買入持有」幾多（打和 = 0 分），輔以路徑穩唔穩

**分數高 → 較容易生仔傳去下一代。**

### 4. 一代之內發生咩事？

好似一班人考試完再收生：

```
一班學生考試
    ↓
分數高嘅幾個「保送」直接入下一代（精英）
    ↓
其餘多數位：抽兩個家長 → 合併筆記 → 加少少突變 → 成新學生
    ↓
最後一個位塞一個「外來插班生」（保持新鮮諗法）
    ↓
成新一班，再考
```

#### 抽家長（錦標賽）

每次隨機抽 3 個學生出嚟比併，分數最高嗰個做阿爸／阿媽。  
差生都有少少機會（撞啱同更差嘅同組）——唔好殺晒探索。  
好處：就算全班分數好接近，「邊個叻啲」照樣分得出，選擇壓力唔會後期消失。

#### 合併筆記（雜交）

仔女每一格筆記，50% 機會抄阿爸、50% 抄阿媽。  
好似兩本溫書各撕一半砌埋。

#### 突變

合併完之後，有機會改亂幾格：

- 多數（約 80%）：喺原數上面抖少少
- 少數（約 20%）：整格重寫

用意：避免成班人永遠困喺同一個壞習慣。

#### 精英

最好嗰幾個 **原封不動** 帶落下一代。  
唔好因為運氣差，把今代冠軍弄丟。

#### 插班生

每代最後一個位，換一個新面孔。  
股票頁特別啲：插班生 **只重抽「指標點樣調」**，大腦決策部分可以沿用——持續試新週期／門檻，又唔使次次打亂已經學識嘅買賣習慣。

### ① 貪食蛇（`/snake`）

**要學咩：** 唔撞牆／自己、盡量食果。

#### 進化設定（預設）

| 項目     | 數值 | 白話                    |
| -------- | ---- | ----------------------- |
| 族群大小 | 36   | 每代 36 套蛇策略        |
| 突變率   | 0.12 | 約 12% 機會改某一格筆記 |
| 突變幅度 | 0.24 | 改動時抖幾大力          |
| 精英比例 | 0.08 | 約保留 2–3 個直接落代   |
| 播放速度 | 5    | 預設全速                |

#### 大腦結構

```
10 個輸入 → 12 個隱藏神經元 → 3 個輸出
```

| 項目       | 內容                                                                 |
| ---------- | -------------------------------------------------------------------- |
| 基因數量   | **171** 個數字（全部係大腦權重／偏差）                               |
| 輸入（10） | 前／左／右有冇危險、食物相對位置 Δx/Δy、而家面朝邊（上右下左）、身長 |
| 輸出（3）  | 左轉 / 直行 / 右轉（揀最大嗰個）                                     |

#### 分數權重（fitness）

每個策略用 **兩個固定隨機種子**（137、911）各打一場，再 **取平均**（減少單場運氣）。

一場分數：

| 項目         | 權重／公式                            | 白話                   |
| ------------ | ------------------------------------- | ---------------------- |
| 食果（主要） | `食到數量² × 180`                     | 食得愈多，獎勵飆得愈勁 |
| 食果（額外） | `食到數量 × 90`                       | 再加一層鼓勵           |
| 存活         | `行咗幾步 × 0.28`                     | 唔好一早撞死           |
| 接近食物     | 靠近 **+0.16**；遠離 **−0.2**（每步） | 早期教「要行近果」     |

```
一場分數 ≈ 食到²×180 + 食到×90 + 步數×0.28 + 靠近／遠離加減分
總分 = (種子137一場 + 種子911一場) ÷ 2
```

其他規則：格 **20×20**；太耐唔食會餓死（約 `110 + 已食×16` 步）；最多 **10,000** 步。

### ② 打磚塊（`/breaker`）

**要學咩：** 板跟波、接住、清磚。

#### 進化設定（預設）

| 項目     | 數值 | 白話                 |
| -------- | ---- | -------------------- |
| 族群大小 | 12   | 物理重，所以學生少啲 |
| 突變率   | 0.14 | 比蛇稍高             |
| 突變幅度 | 0.26 | 抖動稍大             |
| 精英比例 | 0.10 | 約保留 1 個精英      |
| 播放速度 | 5    | 預設全速             |

#### 大腦結構

```
8 個輸入 → 12 個隱藏神經元 → 3 個輸出
```

| 項目      | 內容                                               |
| --------- | -------------------------------------------------- |
| 基因數量  | **147** 個數字                                     |
| 輸入（8） | 板 X、球 X/Y、球速 X/Y、最近磚相對位置、剩餘磚比例 |
| 輸出（3） | 向左 / 停 / 向右                                   |

#### 分數權重（fitness）

每個策略用 **五場真·隨機場景**（`Math.random`：發球角、板位、球位、磚格微移、撞板／撞磚抖動）各打一場，再 **取平均**——逼 AI 真係識跟波，唔好背死一條路線。  
畫面循環重播 **每圈再抽一場**，用家會睇到唔同版本。

一場分數：

| 項目       | 權重                                      | 白話                           |
| ---------- | ----------------------------------------- | ------------------------------ |
| 清磚       | 每磚 **× 140**                            | 主力：清得多先係王道           |
| 全清獎勵   | 清晒全部磚 **+2000**                      | 大獎                           |
| 全清速度   | `(剩時間比例)² × 2200 + 剩時間比例 × 500` | **愈快清晒分差愈大**（二次方） |
| 消磚效率   | `(清磚數 / 步數) × 1400`                  | 未全清都獎「消得密」           |
| 跟波塑造   | 波喺下半場對準球（有上限）                | 早期學跟波；唔可以 chok 過清磚 |
| 多餘接波罰 | 超出「清磚×2+4」每次 **−6**               | **禁止**靠 hit 數無限接波刷分  |

```
一場分數 ≈ 清磚×140
         +（全清 ? 2000 + 速度獎 : 0）
         + 消磚效率
         + 跟波（有 cap）
         − 多餘接波×6
總分 = (三個發球角分數) ÷ 3
```

例：同樣全清，**300 步完成** 遠高於 **3000 步完成**（速度獎可以差過千）。  
唔獎「接波次數／拖時間」——否則 AI 會 chok hit 而唔消磚。

物理：場約 **560×420**；板闊 **92**；球速會夾喺約 **4–7**；最多 **3,600** 步。

### ③ 股票交易（`/stock`）

**要學咩：** 點調技術指標參數、幾時買／持／賣。

#### 進化設定（預設）

| 項目     | 數值 | 白話                                |
| -------- | ---- | ----------------------------------- |
| 族群大小 | 150  | 每代 150 套策略（基因多，要夠人搜） |
| 突變率   | 0.12 | 基數；頭／尾再乘倍數（見下）        |
| 突變幅度 | 0.22 | 基數；頭／尾再乘倍數                |
| 精英比例 | 0.08 | 約保留 2–3 個精英                   |
| 播放速度 | 5    | 預設全速（代間唔刻意等人）          |
| 神經網絡 | 開   | 可關 → 改用規則投票                 |

#### 基因體分兩邊（好重要）

總共約 **310** 個數字 = **17 個頭** + **293 個大腦權重**。

| 區段 | 數量 | 內容                          | 突變（相對基數）               |
| ---- | ---- | ----------------------------- | ------------------------------ |
| 頭   | 17   | 指標週期、RSI／威廉買賣門檻等 | 機率 **×3**、幅度 **×1.5**     |
| 尾   | 293  | 決策大腦權重                  | 機率 **×0.55**、幅度 **×0.55** |

白話：**多用時間試「點調參數」**；大腦仍然改得輕過參數，但夠力離開「每日翻倉 thrash」死角。  
插班生：只重抽頭（參數），保留尾。

#### 大腦結構（神經網絡開時）

```
21 個輸入 → 10 個隱藏 → 5 個隱藏 → 3 個輸出（買／持／賣）
```

21 維大致：高低開收（K 線結構／日回報）、快慢均線、威廉、ROC、RSI、MACD、保力加、波動、成交量、N 日新高、持倉、同 RSI／威廉距離門檻……  
全部指標常開；持倉狀態永遠開。

#### 網絡點樣落倉（神經網絡開時）

輸出係 **買／持／賣** 三個 tanh 分數，唔係亂 argMax 就郁：

| 規則                           | 白話                     |
| ------------------------------ | ------------------------ |
| 買要 **贏 hold ≥ 0.08** 先開多 | 貼身分數唔好每日跳       |
| 賣要 **贏 hold ≥ 0.08** 先平倉 | 同上                     |
| **做多最少 5 根 K** 先畀沽     | 唔好拿住幾日就走         |
| **空倉最少 5 根 K** 先畀再買   | **堵截噚日沽、今日買返** |

規則模式冇用呢兩樣（本身已有趨勢閘，較少抖倉）。

#### 固定用嘅指標

高低開收、移動平均線、威廉、ROC、RSI、MACD、保力加通道、波動率、成交量、N 日新高。

#### 分數權重（fitness）

- 只用時間 **前 80%** 做訓練分數；**後 20% 測試唔入收生**
- 每筆買賣成本 **0.1%**（`0.001`）
- 起始虛擬本金 **10,000**

```
總分 ≈ 全段分數×0.90 + 半段穩健分數×0.10
        − 大腦 L2 罰（權重平均平方 × 0.28）
```

**一段分數（全段同每一半都用呢套）——超額回報優先，兼推高實際持倉回報：**

主軸係 `log(策略最終資產 ÷ 死買唔賣最終資產)`——同大市打和 = 0 分，  
所以淨靠「跟大市」攞唔到超額分，**贏大市先有主力分**。  
另外用絕對回報、平均持倉、換手率、空倉罰，避免牛市 thrash／成世坐現金。

| 項目                       | 權重          | 白話                                         |
| -------------------------- | ------------- | -------------------------------------------- |
| 年化超額（log excess／年） | × **220**     | 每年贏大市幾多（主力）                       |
| 累積超額（log excess）     | × **45**      | 成段合共贏幾多                               |
| 累積回報                   | × **55**      | 推高實際訓練回報，唔好淨打和超額             |
| 平均持倉（mean exposure）  | × **24**      | 牛市肯入市、少坐現金                         |
| Sharpe                     | × **5**       | 穩陣路徑有分                                 |
| 最大回撤                   | × **−10**     | 跌得深扣分                                   |
| 平均換手（mean turnover）  | × **−18**     | 罰每日全進全出（fee 流血）；比舊版輕，留呼吸 |
| 空倉罰                     | **−40 / −12** | 持倉比例好低：約低於 8% 重罰、低於 20% 輕罰  |

**半段穩健：** 訓練再切兩半，  
`0.22 × 較差半 + 0.78 × 兩半平均`  
（一個半段差唔會完全打死）。

開局會塞幾個經典種子策略（第一個接近「死買唔賣」：強買、難賣），  
唔使由完全亂權重慢慢爬。

> **期望管理：** 日線只做多 + 0.1% 成本，**穩定打贏 QQQ 本身好難**。  
> Demo 成功可以係：有合理買賣節奏、唔 thrash、訓練唔輸太多 buy-and-hold——唔係保證「印鈔機」。

#### 規則模式（關神經網絡時）

唔用 293 個權重；改為：

- SMA／MACD／RSI／威廉 **多數贊成**（≥2 票）→ 買
- **賣出有趨勢閘：**
    - 上升趨勢（快線 > 慢線）：要 **RSI 同威廉一齊過熱** 先賣
    - 否則：RSI **或** 威廉過熱 → 賣

#### 畫面額外顯示

- 訓練／測試回報、死買唔賣對照
- 冠軍指標參數
- **匯出 Pine Script**（TradingView）
- **匯出富途 Python**（富途牛牛自編指標；含 state machine + F16 持倉回饋）

### ④ 蒙地卡羅（`/stock-mc`）——用「亂試」搵炒法

先用一句講晒：

> **股票交易頁** 同 **蒙地卡羅頁**，考嘅係 **同一張卷**（幾時買、幾時賣、指標點調）。  
> 分別只係：**點樣搵好答案** 唔同。

| 頁面                    | 好似做咩                                                                       |
| ----------------------- | ------------------------------------------------------------------------------ |
| 股票交易（`/stock`）    | 好學生生仔：叻嘅人結婚、仔女抄筆記再亂改少少                                   |
| 蒙地卡羅（`/stock-mc`） | 買六合彩式亂試：一次抽好多張「做法紙」，計分，**最好嗰張永遠留低**，下一輪再抽 |

「蒙地卡羅」個名聽落好大路——其實就係 **靠大量亂試去估答案** 嘅老方法（賭場、天氣預報都有人用）。  
呢度唔係教你賭錢，只係借呢種 **「試好多、留最好」** 嘅精神。

#### 想像：煲湯試味

你想煲一煲好湯，唔知鹽落幾多、火候點先啱：

1. **一次試 40 煲**（每批樣本數，預設 40）
2. 其中大約 **三分一**，係抄而家最好嗰煲，再 **加減少少鹽**（局部探索——喺冠軍附近再磨）
3. 其餘 **三分二**，完全亂嚟：鹽多鹽少、有人落薑有人唔落（全域亂試）
4. 每煲都「試味」打分（分數計法同股票交易頁 **一模一樣**）
5. 如果有煲贏過舊冠軍 → **換成新冠軍**；舊冠軍唔會因為運氣差就掉咗
6. 再煲下一批 40 煲……重複幾百輪

第一批仲會塞幾煲「經典菜式」（例如常見均線做法），唔好一開始完全亂嚟。

#### 同「生仔進化」差喺邊？

|                  | 股票交易頁（生仔）       | 蒙地卡羅頁（亂試）                                                   |
| ---------------- | ------------------------ | -------------------------------------------------------------------- |
| 點搵下一個做法   | 兩個叻學生生仔，合併筆記 | **唔生仔**；直接亂抽新紙                                             |
| 會唔會抄冠軍？   | 有突變、有精英           | 有——**抄冠軍再改少少**（側欄叫「局部探索比例」）                     |
| 計分卷           | 同一套                   | **同一套**                                                           |
| 畫面睇到咩       | 圖表、買賣、回報         | **一樣**；只係計數器寫「批次」唔係「世代」                           |
| 側欄「幾多人試」 | 叫「族群大小」           | 叫「**每批樣本數**」                                                 |
| 側欄「亂改幾多」 | 叫「突變率」             | 叫「**局部探索比例**」（愈高＝愈多喺冠軍附近磨；愈低＝愈多完全亂試） |

#### 側欄預設（扭掣用）

| 掣           | 預設 | 白話                                 |
| ------------ | ---- | ------------------------------------ |
| 每批樣本數   | 40   | 每一輪試幾多套做法                   |
| 局部探索比例 | 0.35 | 約 35% 抄冠軍再改；其餘完全亂嚟      |
| 播放速度     | 5    | 5 = 跑到盡快                         |
| 神經網絡     | 開   | 關咗就用簡單規則投票（同股票交易頁） |

分數、指標參數、訓練／測試、匯出 Pine／富途 Python——**全部同 ③ 股票交易頁共用**，唔使學第二套。  
兩邊嘅「冠軍筆記」亦可以互相匯入匯出。

#### 點玩（跟住撳就得）

1. 側欄撳 **蒙地卡羅**（或開網址 `/stock-mc`）
2. 等股票數據 load 完（預設 QQQ）
3. 撳 **開始**——睇「批次」一路加、最佳分數有冇爬高
4. 想穩陣：把「局部探索比例」**扭高**（多啲喺好做法附近微調）
5. 想撞大運：把「局部探索比例」**扭低**（多啲完全亂試）
6. 撳 **暫停** 睇冠軍點買賣；想對打就另開一個 tab 去 **股票交易** 頁，大家考同一張卷

## 股票頁：點解有「訓練」同「測試」？

想像你溫書：

- **前 80% 日子** = 溫習卷（訓練）  
  電腦用呢段時間計分數、揀邊個學生生仔。
- **後 20% 日子** = 真正考試（測試）  
  **唔參與收生**。只係俾你睇：溫習好嘅人，考生卷係咪一樣叻。

如果訓練好靚、測試又仆，好多時係 **背熟舊卷、唔係真本事**（過度擬合）。  
呢個正正係股票 demo 想教嘅一課。  
**股票交易** 同 **蒙地卡羅** 兩頁用 **同一套溫習卷／生卷切法、同一套計分**，所以可以公平比較：邊種搵答案方法叻啲。

## 常見問題

### 點解股票訓練好慢？

因為每一套做法，都要喺好多年股價上面「假裝交易」一次。  
一次試愈多人／愈多套、歷史愈長，愈慢。

想快：

- 播放速度拉去 **5**
- 「族群大小」或「每批樣本數」細少少
- 試下關神經網絡（改用簡單規則）

### 點解訓練回報靚、測試又仆？

好正常。  
等於背熟溫習卷，生卷唔識。  
記住：**測試段唔入訓練**，就係要揭穿呢種假叻。

### 「生仔」同「蒙地卡羅」邊個好？

**冇一定——呢個就係想你自己試。**

- **生仔（股票交易頁）：** 兩個半靚做法砌埋，有時穩、有時困死
- **蒙地卡羅：** 唔砌人，就係 **亂試 + 抄冠軍再改**；好易明，有時撞到寶，有時嘈

同一張分卷，開兩個分頁一齊跑，睇測試回報同參數設定，自己判斷。

### 搵到「必勝參數」未？

**未，亦唔應該咁諗。**  
呢啲頁示範嘅係「電腦點樣自己試錯搵做法」，**唔係保證未來賺錢**。

### 匯出 Pine Script／富途 Python 係咩？

冠軍出嚟之後，喺「最佳指標參數」旁可以：

| 掣                   | 產出    | 用喺邊                   |
| -------------------- | ------- | ------------------------ |
| **匯出 Pine Script** | `.pine` | TradingView strategy     |
| **匯出富途 Python**  | `.py`   | 富途牛牛自編 Python 指標 |

兩邊都由同一份冠軍基因體生成（參數 + 網絡權重／規則）。  
富途版用 **state machine**（空倉先開、有倉先平）+ **F16 持倉回饋** 餵網絡，畫 SMA／BB／N 日高 + 買賣 icon。  
一樣：**只供學習，唔係實盤建議。**  
股票交易頁同蒙地卡羅頁共用；冠軍檔案兩邊可以互用。

### 點解股票「train 唔出必勝模型」？

好正常，原因包括：

1. **日線只做多 + 0.1% 成本**，穩定打贏 QQQ buy-and-hold 本身好難
2. Fitness 主軸係 **超額回報**——跟大市 ≈ 0 分，唔係「賺到錢就滿分」
3. 神經網絡基因多，GA 無 backprop，容易卡喺 thrash 或空倉
4. 訓練靚、測試仆 = 過擬合（溫習卷 vs 生卷）

想穩陣啲可以：**關神經網絡** 用規則模式；或接受「合理節奏、少 thrash」當 demo 成功，唔好當印鈔機。
