# EvoLab 是什麼？

用一句話概括：**教電腦自行試錯，學習做決定。**

不是人手編寫「見到紅燈就停」這類死規則，而是：

1. 隨機產生一批「做法」
2. 觀察哪個表現較好
3. 好的保留並繁殖
4. 後代有少量突變（略作改動）
5. 重複數百輪

主線做法稱為 **遺傳演算法**（英文 Genetic Algorithm）——自然界以此進化動植物；此處則用來進化「玩遊戲／模擬炒股」的方法。  
股票另有一頁 **蒙地卡羅**（網址 `/stock-mc`）：考題相同，但尋找答案的方法不同——**隨機試大量做法，保留最好的，再試多輪**。

首頁 **演算法原理**（`/theory`）以圖示與小實驗，解釋這套「繁殖進化」如何運作、以及本專案如何實踐。

> **重要：** 本專案為 **教學示範**，並非投資軟件。股票頁僅示範演算法，**並非投資建議**。  
> **請勿當作提款機。**

## 核心概念（生活化版）

### 1. 族群（一班學生）

每一代有數十個「學生」（例如 36 個）。  
每個學生代表 **一套做法**。  
專業名稱：**族群（population）**。

### 2. 基因體（學生的筆記）

每個學生身上有一串數字，稱為 **基因體**（英文 genome）。  
可想像成考試溫書筆記：

- 貪食蛇／打磚塊／Flappy Bird：筆記 = 大腦中「見到什麼就怎麼做」的參數
- 股票：筆記 = **指標如何調整** + **如何決定買賣**

### 3. 適應度（考試分數）

每個學生考一次試，取得一個 **分數**（專業名稱 fitness）：

- 蛇：吃到多少、走了多久
- 打磚塊：清了多少磚、清得夠不夠快
- Flappy Bird：過了多少條水管、飛了多久
- 股票：有否喺多個 walk-forward validation 時段贏過「一路買入持有」；亂買賣、長期空倉會扣分；最後 20% 封存測試唔入分

**分數高 → 較容易繁殖傳到下一輪。**

### 4. 一代之內發生什麼事？

如同考試完畢再招生：

```
一班學生考試
    ↓
分數高的幾個「保送」直接進入下一代（精英）
    ↓
其餘多數位：抽兩個家長 → 合併筆記 → 加少量突變 → 成為新學生
    ↓
最後一個位放入一個「外來插班生」（保持新鮮想法）
    ↓
成為新一班，再考
```

#### 抽家長（錦標賽）

每次隨機抽 3 個學生比較，分數最高者成為父母。  
差生也有少量機會（剛好與更差的同組）——不要殺光「嘗試新事物」的可能。  
好處：即使全班分數很接近，「誰較出色」仍分得出來。

專業名稱：**錦標賽選擇（tournament selection）**。

#### 合併筆記（雜交）

後代每一格筆記，一半機會抄父親、一半機會抄母親。  
如同兩本筆記各撕一半拼合。

專業名稱：**雜交（crossover）**。

#### 突變

合併完之後，有機會改亂幾格：

- 多數（約 80%）：在原數上略作變動
- 少數（約 20%）：整格重寫

用意：避免整班人永遠困在同一個壞習慣。

專業名稱：**突變（mutation）**。

#### 精英

最好的幾個 **原封不動** 帶入下一代。  
免得因一時運氣差，無故失去今代冠軍。

專業名稱：**精英保留（elitism）**。

#### 插班生

每代最後一個位，換一個新面孔。  
股票頁較特別：插班生 **只重抽「指標如何調整」**，大腦決策部分可沿用——持續試新週期／門檻，又不必每次打亂已學到的買賣習慣。

專業名稱：**移民（immigrant）**。

以下為 **程式預設**（側欄可改「多少人一起試」、亂改程度、速度；股票兩頁預設開神經網絡，亦可關返規則模式）。

### ① 貪食蛇（`/snake`）

**要學什麼：** 不撞牆／自己、盡量吃果實。

#### 進化設定（預設）

| 項目     | 數值 | 說明                    |
| -------- | ---- | ----------------------- |
| 族群大小 | 36   | 每代 36 套蛇策略        |
| 突變率   | 0.12 | 約 12% 機會改某一格筆記 |
| 突變幅度 | 0.24 | 改動時變動幅度          |
| 精英比例 | 0.08 | 約保留 2–3 個直接落代   |
| 播放速度 | 5    | 預設全速                |

#### 大腦結構

```
10 個輸入 → 12 個中間神經元 → 3 個輸出
```

| 項目       | 內容                                                         |
| ---------- | ------------------------------------------------------------ |
| 基因數量   | **171** 個數字（全部是大腦內的參數）                         |
| 輸入（10） | 前／左／右有否危險、食物方向（左右上下距離）、當前面朝、身長 |
| 輸出（3）  | 左轉 / 直行 / 右轉（選分最高的）                             |

#### 分數權重（fitness）

每個策略用 **兩個固定運氣編號**（137、911）各打一場，再 **取平均**（減少單場好運）。

一場分數：

| 項目         | 權重／公式                            | 說明                   |
| ------------ | ------------------------------------- | ---------------------- |
| 食果（主要） | `食到數量² × 180`                     | 吃得愈多，獎勵飆得愈高 |
| 食果（額外） | `食到數量 × 90`                       | 再加一層鼓勵           |
| 存活         | `走了幾步 × 0.28`                     | 不要過早撞死           |
| 接近食物     | 靠近 **+0.16**；遠離 **−0.2**（每步） | 早期教「要走近果實」   |

```
一場分數 ≈ 食到²×180 + 食到×90 + 步數×0.28 + 靠近／遠離加減分
總分 = (編號137一場 + 編號911一場) ÷ 2
```

其他規則：格 **20×20**；太久不吃會餓死（約 `110 + 已食×16` 步）；最多 **10,000** 步。

### ② 打磚塊（`/breaker`）

**要學什麼：** 板跟球、接住、清磚。

#### 進化設定（預設）

| 項目     | 數值 | 說明                   |
| -------- | ---- | ---------------------- |
| 族群大小 | 12   | 物理負擔重，故學生較少 |
| 突變率   | 0.14 | 比蛇稍高               |
| 突變幅度 | 0.26 | 改動稍大               |
| 精英比例 | 0.10 | 約保留 1 個精英        |
| 播放速度 | 5    | 預設全速               |

#### 大腦結構

```
8 個輸入 → 12 個中間神經元 → 3 個輸出
```

| 項目      | 內容                                               |
| --------- | -------------------------------------------------- |
| 基因數量  | **147** 個數字                                     |
| 輸入（8） | 板左右位置、球位置／速度、最近磚相對位置、剩餘磚數 |
| 輸出（3） | 向左 / 停 / 向右                                   |

#### 分數權重（fitness）

每個策略用 **五場真·隨機場景**（發球角度、板位、球位、磚格微移、撞板／撞磚抖動）各打一場，再 **取平均**——逼電腦真正學會跟球，而非背死一條路線。  
畫面循環重播 **每圈再抽一場**，你會看到不同版本。

一場分數：

| 項目       | 權重                                      | 說明                         |
| ---------- | ----------------------------------------- | ---------------------------- |
| 清磚       | 每磚 **× 140**                            | 主力：清得多才是王道         |
| 全清獎勵   | 清完全部磚 **+2000**                      | 大獎                         |
| 全清速度   | `(剩時間比例)² × 2200 + 剩時間比例 × 500` | **愈快清完分差愈大**         |
| 消磚效率   | `(清磚數 / 步數) × 1400`                  | 未全清也獎「消得密」         |
| 跟球加分   | 球在下半場對準板（有上限）                | 早期學跟球；不可靠此蓋過清磚 |
| 多餘接球罰 | 超出「清磚×2+4」每次 **−6**               | **禁止**只顧接球、不消磚刷分 |

```
一場分數 ≈ 清磚×140
         +（全清 ? 2000 + 速度獎 : 0）
         + 消磚效率
         + 跟球（有上限）
         − 多餘接球×6
總分 = 幾場隨機場景分數平均
```

例：同樣全清，**300 步完成** 分數遠高於 **3000 步才完成**（速度獎可差過千）。  
不獎「接球次數／拖時間」——否則電腦會只顧滾球不消磚。

物理：場約 **560×420**；板闊 **92**；球速大約夾在 **4–7**；最多 **3,600** 步。

**要學什麼：** 在重力下如何拍翼，穿過水管縫隙。

#### 進化設定（預設）

| 項目     | 數值 | 說明                    |
| -------- | ---- | ----------------------- |
| 族群大小 | 40   | 每代 40 套飛法          |
| 突變率   | 0.13 | 約 13% 機會改某一格筆記 |
| 突變幅度 | 0.24 | 改動時變動幅度          |
| 精英比例 | 0.08 | 約保留 3 個直接落代     |
| 播放速度 | 5    | 預設全速                |

#### 大腦結構

```
6 個輸入 → 10 個中間神經元 → 2 個輸出
```

| 項目      | 內容                                                   |
| --------- | ------------------------------------------------------ |
| 基因數量  | **92** 個數字（全部是大腦參數）                        |
| 輸入（6） | 鳥高度、鳥上下速度、下條管距離、縫隙上下沿、離縫心距離 |
| 輸出（2） | 拍翼 / 滑翔（選分高的）                                |

#### 分數權重（fitness）

每個策略用 **三個固定運氣編號**（41、277、809）各飛一場，再 **取平均**（減少剛好抽到好走水管的運氣成分）。

一場分數：

| 項目       | 權重／公式                         | 說明                   |
| ---------- | ---------------------------------- | ---------------------- |
| 過管（主） | `通過水管數² × 220`                | 過得愈多，獎勵飆得愈高 |
| 過管（額） | `通過水管數 × 80`                  | 再加一層鼓勵           |
| 存活       | `飛了幾步 × 0.35`                  | 不要過早撞死           |
| 對準縫心   | 飛近縫心 **+**、離遠 **−**（每步） | 早期教「對準中間飛」   |

```
一場分數 ≈ 過管²×220 + 過管×80 + 步數×0.35 + 縫心加減分
總分 = (編號41 + 編號277 + 編號809) ÷ 3
```

其他規則：場 **360×520**；有重力，拍翼向上；撞地／撞頂／撞管即死；最多 **8,000** 步。  
畫面：藍天綠管、像素黃雀、城市剪影貼地。

### ④ 股票交易（`/stock`）

**要學什麼：** 如何調整技術指標參數、何時買／持／**平倉**（做多或落現金）。

#### 進化設定（預設）

| 項目     | 數值   | 說明                                              |
| -------- | ------ | ------------------------------------------------- |
| 族群大小 | 150    | 每代 150 套策略（筆記多，需夠人一起試）           |
| 突變率   | 0.12   | 基數；頭／尾再乘倍數（見下）                      |
| 突變幅度 | 0.22   | 基數；頭／尾再乘倍數                              |
| 精英比例 | 0.08   | 約保留 2–3 個精英                                 |
| 播放速度 | 5      | 預設全速（代與代之間不刻意等待）                  |
| 神經網絡 | **開** | 預設用決策頭（18→10→5→3）；可關返簡單規則投票模式 |

#### 基因體分兩邊（很重要）

總共約 **281** 個數字 = **18 個頭** + **263 個大腦參數**。

| 區段 | 數量 | 內容                 | 突變（相對基數）                  |
| ---- | ---- | -------------------- | --------------------------------- |
| 頭   | 18   | 指標週期、買賣門檻等 | 改得 **密 3 倍**、幅度 **1.5 倍** |
| 尾   | 263  | 決策大腦參數         | 改得 **較少**（約 0.55 倍）       |

說明：**多用時間試「指標如何調整」**；大腦也改，但改得比參數輕。  
插班生：只重抽頭（參數），保留尾。  
如此可戒掉「日日買完又沽」的壞習慣，又不必每次打亂整個大腦。

#### 大腦結構（預設開啟；可關）

```
18 個輸入 → 10 個中間 → 5 個中間 → 3 個輸出（買／持／平倉）
```

18 樣大致：快慢均線、威廉、ROC、RSI、MACD、保力加、波動、成交量、**N 日新高**、**N 日新低**、現在持倉（做多／空倉）、與買賣門檻差多遠……  
**不餵** 開／高／低／收（K 線結構）。全部指標常開；「現在持倉」永遠開。

冠軍嘅指標參數 + 決策頭權重會寫入瀏覽器；**refresh 後仍會還原重播**，除非按側欄 **重設**。

#### 網絡如何落倉（神經網絡開啟時）

輸出是 **買／持／平倉** 三個分數，不是亂選最大就動。  
倉位 ∈ **做多 / 空倉**（`+1 / 0`）——賣出 = **平倉落現金**，**唔做沽空**。

| 規則                                          | 說明                                 |
| --------------------------------------------- | ------------------------------------ |
| 空倉時，買要 **贏「持／平」至少 0.08** 才做多 | 分數貼身時不要日日跳                 |
| 已做多時，「買」當留守；明確平倉 → **落現金** | 近平手唔亂出                         |
| 空倉時平倉訊號 **唔會** 開沽空                | 淨 long／cash                        |
| 日日亂翻用 **分數罰**（換手罰）               | **沒有**「最少持幾日才准轉倉」的硬鎖 |

關閉神經網絡時，用下面「規則模式」+ 同一套換手罰，一樣 **不硬鎖** 持倉日數。

#### 固定使用的指標

移動平均線、威廉、ROC、RSI、MACD、保力加通道、波動率、成交量、**N 日新高**、**N 日新低**（NN 輸入不再包含開高低收）。

| 指標     | 計法                                          | 大腦看到什麼   |
| -------- | --------------------------------------------- | -------------- |
| N 日新高 | 過去 N 日最高價；收市愈近此高，愈貼「創新高」 | 突破／追高訊號 |
| N 日新低 | 過去 N 日最低價；收市愈近此低，愈貼「創新低」 | 支撐／貼低訊號 |

兩個「望多久」**各自一個數字**（約 10–120 日），進化可分開調整；圖表下拉可選「N 日高」／「N 日低」看線。

#### 成交假設（與複製出去的腳本對齊）

- **今日收市才決定 → 第二日開市才成交**（不是即買即得）
- 每次轉倉單邊成本約 **0.15%** × `|Δ倉位|`（做多↔空倉 = 1 倍成本）
- **沒有**「最少持幾日才准轉倉」硬鎖；日日亂翻靠 **成本 + 分數換手罰** 打壓

#### 分數權重（fitness）

- **前 80% = development**：先留約三年 expanding training history，再逐年向前做 validation
- **尾 20% = 封存 holdout**：完全唔入 fitness；暫停訓練先計同顯示
- 每個 validation window 起始虛擬本金 **10,000**
- 多個 window 一齊揀冠軍，避免只背熟單一次切割

```
總分 ≈ validation folds 中位數×0.50
        + folds 平均×0.30
        + 最差 fold×0.20
        − 大腦參數不要太大（開啟神經網絡時）
```

（每個 fold 用 **log 回報** 同 **相對買入持有超額**；輸大市有額外罰。）

**一段分數——贏大市才是主力；淨跟大市 ≈ 0 分：**

主軸是「你最終有多少錢 ÷ 一路買入持有最終有多少錢」——  
與大市打和 = 0 分，**贏大市才有主力分**。  
另外獎實際賺到、肯入市（做多算曝險）；罰跌得深、日日翻倉、長期空倉。

| 項目           | 權重（大概）    | 說明                                        |
| -------------- | --------------- | ------------------------------------------- |
| 每年贏大市多少 | × **220**       | 主力（log 超額）                            |
| 成段 log 回報  | × **55**        | 輔力（壓縮極端倍數）                        |
| 平均有否入市   | × **30**        | 做多算；少空倉                              |
| 路徑穩不穩     | × **5**         | 少量分                                      |
| 最大回吐       | × **−12**       | 跌得深扣分                                  |
| 平均換手多密   | **−28t − 80t²** | 罰日日買完又沽；愈密罰愈重（二次方）        |
| 長期空倉       | **−50 / −18**   | 入市比例很低：約低於 8% 重罰、低於 25% 輕罰 |

**Walk-forward 穩健：** expanding training history 後面接順序 validation windows；最差 fold 仍佔 20%，唔俾其他升市完全遮住一次大跌。

開局會放入幾個「經典菜式」做法（第一個接近「易買、難沽」），  
不必由完全亂數字慢慢爬。

> **期望管理：** 日線 long／cash + 0.15% 成本 + 次日開盤，**穩定打贏 QQQ 本身很難**。  
> 選冠軍完全唔睇最後 20% holdout；暫停揭卷後先判斷有冇真係泛化。Demo 成功 ≠ 實盤印鈔。

#### 規則模式（側欄關閉神經網絡時）

不用 263 個大腦參數；改為：

- 均線／MACD／RSI／威廉 **多數贊成**（至少 2 票）→ **做多**
- **平倉有趨勢閘**（超買 → 長倉落現金）：
    - 上升趨勢（快線 > 慢線）：要 **RSI 與威廉一齊過熱** 才平倉
    - 否則：RSI **或** 威廉過熱 → 平倉
- 空倉唔開沽空；買票夠先再做多

#### 畫面額外顯示

- Development／封存測試回報、**分段**「一路買入持有」；訓練中封存測試顯示「暫停後揭示」
- 冠軍指標參數（含 N 日新高／新低週期）
- 重播 HUD：做多／空倉；圖表買入／平倉標記
- 圖表可切：**均線+保力加**／動量／MACD／風險／**N 日高**／**N 日低**
- 側欄 **訓練控制** 可以：
    - **複製 Pine Script**（貼到 TradingView；long entry + close）
    - **複製富途 Python**（貼到富途牛牛自編指標）

### ⑤ 蒙地卡羅（`/stock-mc`）——用「亂試」找炒法

先用一句話講完：

> **股票交易頁** 與 **蒙地卡羅頁**，考的是 **同一張卷**（何時做多／平倉、指標如何調）。  
> 分別只是：**如何尋找好答案** 不同。

| 頁面                    | 好比做什麼                                                                     |
| ----------------------- | ------------------------------------------------------------------------------ |
| 股票交易（`/stock`）    | 好學生繁殖：出色的人結合、後代抄筆記再略作亂改                                 |
| 蒙地卡羅（`/stock-mc`） | 買六合彩式亂試：一次抽很多張「做法紙」，計分，**最好那張永遠留下**，下一輪再抽 |

「蒙地卡羅」名稱聽起來很大路——其實就是 **靠大量亂試去估答案** 的老方法（賭場、天氣預報都有人用）。  
此處不是教你賭錢，只是借用這種 **「試很多、留最好」** 的精神。

#### 想像：煲湯試味

你想煲一鍋好湯，不知鹽放多少、火候如何才對：

1. **一次試 40 鍋**（每批樣本數，預設 40）
2. 其中大約 **三分一**，是抄現在最好那鍋，再 **加減少量鹽**（在冠軍附近再磨）
3. 其餘 **三分二**，完全亂來：鹽多鹽少、有人放薑有人不放
4. 每鍋都「試味」打分（分數計法與股票交易頁 **一模一樣**）
5. 如果有鍋贏過舊冠軍 → **換它做新冠軍**；舊冠軍不會因一次運氣差就被扔掉
6. 再煲下一批 40 鍋……重複數百輪

第一批還會放入幾鍋「經典菜式」（例如常見均線做法），不要一開始完全亂來。

#### 與「繁殖進化」差在哪裡？

|                  | 股票交易頁（繁殖）         | 蒙地卡羅頁（亂試）                                                   |
| ---------------- | -------------------------- | -------------------------------------------------------------------- |
| 如何找下一個做法 | 兩個出色學生繁殖，合併筆記 | **不繁殖**；直接亂抽新紙                                             |
| 會否抄冠軍？     | 有突變、有精英             | 有——**抄冠軍再改少少**（側欄稱「局部探索比例」）                     |
| 計分卷           | 同一套                     | **同一套**                                                           |
| 畫面看到什麼     | 圖表、買賣、回報           | **一樣**；只是計數器寫「批次」不是「世代」                           |
| 側欄「多少人試」 | 稱「族群大小」             | 稱「**每批樣本數**」                                                 |
| 側欄「亂改多少」 | 稱「突變率」               | 稱「**局部探索比例**」（愈高＝愈多在冠軍附近磨；愈低＝愈多完全亂試） |

#### 側欄預設（調整用）

| 項目         | 預設   | 說明                                               |
| ------------ | ------ | -------------------------------------------------- |
| 每批樣本數   | 40     | 每一輪試多少套做法                                 |
| 局部探索比例 | 0.35   | 約 35% 抄冠軍再改；其餘完全亂來                    |
| 播放速度     | 5      | 5 = 跑到盡快                                       |
| 神經網絡     | **開** | 預設決策頭；可關返簡單規則投票（與股票交易頁相同） |

分數、指標參數、development／封存測試、複製 Pine／富途 Python——**全部與 ④ 股票交易頁共用**，不必學第二套。
兩邊的「冠軍筆記」亦可互相匯入匯出。

#### 如何遊玩（按步驟操作即可）

1. 側欄按 **蒙地卡羅**（或開啟網址 `/stock-mc`）
2. 等股票數據 load 完（預設 QQQ）
3. 按 **開始**——看「批次」一路增加、最佳分數有否爬高
4. 想穩陣：將「局部探索比例」**調高**（多在好做法附近微調）
5. 想靠運氣：將「局部探索比例」**調低**（多完全亂試）
6. 按 **暫停** 看冠軍如何買賣；想對打就另開一個 tab 去 **股票交易** 頁，大家考同一張卷

## 股票頁：為什麼有 development 與封存測試？

想像你有多份練習卷同一份密封正式卷：

- **前約 80% 日子 = development**
  先累積 training history，再順住時間做多個 validation folds。GA／蒙地卡羅只可根據呢啲 folds 揀冠軍。
- **最後約 20% 日子 = 封存 holdout**
  完全唔參與 fitness；訓練期間連策略曲線同回報都唔顯示，按暫停先揭示一次。

每個 validation fold 都會計相對買入持有超額；總分同時睇中位、平均同最差一段。咁仍然唔代表未來必勝，但至少唔會再一邊叫「測試」，一邊每代偷睇答案。
**股票交易** 與 **蒙地卡羅** 兩頁共用同一套 walk-forward／holdout 規則，所以可以公平比較搜尋方法。

## 常見問題

### 為什麼股票訓練很慢？

因為每一套做法，都要在很多年股價上「假裝交易」一次。  
一次試愈多人／愈多套、歷史愈長，愈慢。

想快：

- 播放速度拉到 **5**
- 「族群大小」或「每批樣本數」細少少
- 試試關閉神經網絡（改用簡單規則模式，搜尋空間細啲）

### 為什麼 development 好看、封存測試又跌倒？

很正常。  
等於多份練習卷都做得好，正式卷仍然可以撞到另一種市況。
記住：**封存測試完全唔入總分**；揭卷後表現差，應當視為泛化失敗，唔好再反覆用同一 holdout 調參。

### 「繁殖」與「蒙地卡羅」哪個好？

**沒有一定——這正是想你自己試。**

- **繁殖（股票交易頁）：** 取兩個較好的做法拼合，有時穩、有時困死
- **蒙地卡羅：** 不拼合，就是 **亂試 + 抄冠軍再改**；很容易理解，有時撞到寶，有時飄忽

同一張分卷，開兩個分頁一起跑；各自暫停後再比較封存測試回報同參數設定。

### 找到「必勝參數」了嗎？

**未，亦不應該這樣想。**  
這些頁示範的是「電腦如何自行試錯找做法」，**並非保證未來賺錢**。

### 重新整理之後，冠軍／神經網絡權重去咗邊？

仍喺瀏覽器 **`localStorage`**（key：`evolab-state-v1`）。  
股票兩頁會喺 market data load 完之後，用存低嘅基因體 **重建 replay**——參數表、決策頭、圖表訊號應會返嚟。  
只有按側欄 **重設**（或清瀏覽器網站資料）先會抹走該實驗嘅冠軍同設定。  
各 lab 分開存；重設蛇唔會清股票。

### 複製 Pine Script／富途 Python 是什麼？

冠軍出現之後，在右側 **訓練控制** 可以（內容複製到剪貼簿，不再下載檔）：

| 項目                 | 產出                 | 用於哪裡                 |
| -------------------- | -------------------- | ------------------------ |
| **複製 Pine Script** | Pine 全文 → 剪貼簿   | TradingView 圖表策略     |
| **複製富途 Python**  | Python 全文 → 剪貼簿 | 富途牛牛自編 Python 指標 |

兩邊都由同一份冠軍筆記生成（參數 + 大腦／規則）。  
匯出與瀏覽器模擬對齊：**做多 / 平倉落現金**（唔做沽空）、次日開盤、0.15% 成本；持倉狀態餵回大腦；畫均線／保力加／**N 日高／N 日低** + 買入／平倉標記。  
一樣：**只供學習，並非實盤建議。**  
股票交易頁與蒙地卡羅頁共用；冠軍檔案兩邊可以互用。  
（如果舊檔案是很舊版本的大腦結構，可能匯入不到，要重新訓練一次。）

### 為什麼股票「train 不出必勝模型」？

很正常，原因包括：

1. **日線 long／cash + 0.15% 成本 + 次日開盤**，穩定打贏 QQQ「一路買入持有」本身很難
2. 分數主軸是 **贏大市多少**——跟大市 ≈ 0 分，不是「賺到錢就滿分」
3. 大腦參數很多，進化又沒有「倒數教」（backprop），容易卡在日日亂翻或者長期空倉
4. Development 好看、holdout 跌倒 = 仍然過度擬合——封存段唔入 selection，會如實揭示呢種失敗

現在已加：**walk-forward folds**、**尾 20% 封存 holdout**、**log 回報＋輸大市罰**、**預設十年日線**、**預設神經網絡**、**做多／平倉（唔沽空）**、**冠軍 localStorage 持久化**。
選冠軍只對齊 development validation 超額；holdout 只用嚟做最後樣本外檢查。

## 一句收尾

EvoLab = **亂試一堆做法 → 打分 → 留好的 → 再試**。

- **蛇、磚、Flappy、股票交易：** 類似繁殖——出色的傳下去，後代會略有不同

- **演算法原理：** 圖文解釋整條鏈，再連到各個實驗

學到「電腦如何自行找答案」就夠——**開心即可，請勿當作提款密碼。**
