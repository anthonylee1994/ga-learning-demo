# EvoLab 係咩？

用一句講：**教電腦自己試錯，學做決定。**

唔係人手寫「見到紅燈就停」咁啲死規則，而係：

1. 隨機出一班「策略」
2. 睇邊個做得好
3. 好嘅留下來生仔
4. 仔有少少突變
5. 重複幾百代

呢套做法叫 **遺傳演算法（Genetic Algorithm）**。  
自然界進化物種；呢度進化「玩遊戲／炒股票」嘅方法。

> **重要：** 呢個專案係 **教學 demo**，唔係投資軟件。股票頁只係示範演算法，**唔係投資建議**。

## 核心概念（生活化版）

### 1. 族群（一班學生）

每一代有幾十個「學生」（例如 36 個）。  
每個學生代表 **一套做法**。

### 2. 基因體（學生嘅筆記）

每個學生身上有一串數字，叫 **基因體（genome）**。  
可以想像成考試溫書筆記：

- 貪食蛇／打磚塊：筆記 = 大腦入面「見到咩就點做」嘅參數
- 股票：筆記 = **用邊啲技術指標** + **指標點調** + **點決定買賣**

### 3. 適應度（考試分數）

每個學生考一次試，攞一個 **分數（fitness）**：

- 蛇：食到幾多、行得幾耐
- 打磚塊：清幾多磚、接住幾耐
- 股票：訓練期賺幾多、穩唔穩、會唔會成日空手坐住

**分數高 → 較容易生仔傳去下一代。**

### 4. 一代之內發生咩事？

好似一班人考試完再收生：

```
一班學生考試
    ↓
分數高嘅幾個「保送」直接入下一代（精英）
    ↓
其餘多數位：抽兩個家長 → 合併筆記 → 加少少突變 → 成新學生
    ↓
最後一個位塞一個「外來插班生」（保持新鮮諗法）
    ↓
成新一班，再考
```

#### 抽家長（輪盤）

分數愈高，輪盤上佔位愈大，愈容易被抽中做阿爸阿媽。  
差生都有少少機會——唔好殺晒探索。

#### 合併筆記（雜交）

仔女每一格筆記，50% 機會抄阿爸、50% 抄阿媽。  
好似兩本溫書各撕一半砌埋。

#### 突變

合併完之後，有機會改亂幾格：

- 多數（約 80%）：喺原數上面抖少少
- 少數（約 20%）：整格重寫

用意：避免成班人永遠困喺同一個壞習慣。

#### 精英

最好嗰幾個 **原封不動** 帶落下一代。  
唔好因為運氣差，把今代冠軍弄丟。

#### 插班生

每代最後一個位，換一個新面孔。  
股票頁特別啲：插班生 **只重抽「用邊啲指標／點樣調」**，大腦決策部分可以沿用——持續試新指標組合，又唔使次次打亂已經學識嘅買賣習慣。

## 三個實驗：每個 page 用咗咩參數／權重？

下面係 **程式預設**（側欄可以改族群、突變率、速度；股票仲可以關神經網絡）。  
「權重」= 計分時邊樣加幾多、扣幾多。

---

### ① 貪食蛇（`/snake`）

**要學咩：** 唔撞牆／自己、盡量食果。

#### 進化設定（預設）

| 項目     | 數值 | 白話                    |
| -------- | ---- | ----------------------- |
| 族群大小 | 36   | 每代 36 套蛇策略        |
| 突變率   | 0.12 | 約 12% 機會改某一格筆記 |
| 突變幅度 | 0.24 | 改動時抖幾大力          |
| 精英比例 | 0.08 | 約保留 2–3 個直接落代   |
| 播放速度 | 5    | 預設全速                |

#### 大腦結構

```
10 個輸入 → 12 個隱藏神經元 → 3 個輸出
```

| 項目       | 內容                                                                 |
| ---------- | -------------------------------------------------------------------- |
| 基因數量   | **171** 個數字（全部係大腦權重／偏差）                               |
| 輸入（10） | 前／左／右有冇危險、食物相對位置 Δx/Δy、而家面朝邊（上右下左）、身長 |
| 輸出（3）  | 左轉 / 直行 / 右轉（揀最大嗰個）                                     |

#### 分數權重（fitness）

每個策略用 **兩個固定隨機種子**（137、911）各打一場，再 **取平均**（減少單場運氣）。

一場分數：

| 項目         | 權重／公式                            | 白話                   |
| ------------ | ------------------------------------- | ---------------------- |
| 食果（主要） | `食到數量² × 180`                     | 食得愈多，獎勵飆得愈勁 |
| 食果（額外） | `食到數量 × 90`                       | 再加一層鼓勵           |
| 存活         | `行咗幾步 × 0.28`                     | 唔好一早撞死           |
| 接近食物     | 靠近 **+0.16**；遠離 **−0.2**（每步） | 早期教「要行近果」     |

```
一場分數 ≈ 食到²×180 + 食到×90 + 步數×0.28 + 靠近／遠離加減分
總分 = (種子137一場 + 種子911一場) ÷ 2
```

其他規則：格 **20×20**；太耐唔食會餓死（約 `110 + 已食×16` 步）；最多 **10,000** 步。

### ② 打磚塊（`/breaker`）

**要學咩：** 板跟波、接住、清磚。

#### 進化設定（預設）

| 項目     | 數值 | 白話                 |
| -------- | ---- | -------------------- |
| 族群大小 | 12   | 物理重，所以學生少啲 |
| 突變率   | 0.14 | 比蛇稍高             |
| 突變幅度 | 0.26 | 抖動稍大             |
| 精英比例 | 0.10 | 約保留 1 個精英      |
| 播放速度 | 5    | 預設全速             |

#### 大腦結構

```
8 個輸入 → 12 個隱藏神經元 → 3 個輸出
```

| 項目      | 內容                                               |
| --------- | -------------------------------------------------- |
| 基因數量  | **147** 個數字                                     |
| 輸入（8） | 板 X、球 X/Y、球速 X/Y、最近磚相對位置、剩餘磚比例 |
| 輸出（3） | 向左 / 停 / 向右                                   |

#### 分數權重（fitness）

每個策略用 **三個固定發球角度**（0.72、0.86、1.0）各打一場，再 **取平均**。  
畫面重播用 **0.86** 嗰個角度。

一場分數：

| 項目     | 權重                                     | 白話                 |
| -------- | ---------------------------------------- | -------------------- |
| 清磚     | 每磚 **× 110**                           | 主力：清得多先係王道 |
| 接波次數 | 每次 **× 9**                             | 鼓勵穩接             |
| 跟波塑造 | 波喺下半場時，板對準球最多每步 **+0.08** | 早期學「跟住波」     |
| 存活步數 | 每步 **× 0.01**                          | 好細，淨係打和局用   |
| 全清獎勵 | 清晒全部磚 **+2000**                     | 大獎                 |

```
一場分數 ≈ 清磚×110 + 接波×9 + 跟波塑造 + 步數×0.01 +（全清 ? 2000 : 0）
總分 = (三個發球角分數) ÷ 3
```

物理：場約 **560×420**；板闊 **92**；球速會夾喺約 **4–7**；最多 **10,000** 步。

### ③ 股票交易（`/stock`）

**要學咩：** 用邊啲技術指標、點調參數、幾時買／持／賣。

#### 進化設定（預設）

| 項目     | 數值 | 白話                         |
| -------- | ---- | ---------------------------- |
| 族群大小 | 36   | 每代 36 套策略               |
| 突變率   | 0.12 | 基數；頭／尾再乘倍數（見下） |
| 突變幅度 | 0.22 | 基數；頭／尾再乘倍數         |
| 精英比例 | 0.08 | 約保留 2–3 個精英            |
| 播放速度 | 5    | 預設全速（代間唔刻意等人）   |
| 神經網絡 | 開   | 可關 → 改用規則投票          |

#### 基因體分兩邊（好重要）

總共約 **279** 個數字 = **26 個頭** + **253 個大腦權重**。

| 區段 | 數量 | 內容                          | 突變（相對基數）               |
| ---- | ---- | ----------------------------- | ------------------------------ |
| 頭   | 17   | 指標週期、RSI／威廉買賣門檻等 | 機率 **×3**、幅度 **×1.5**     |
| 頭   | 9    | 指標開／關（mask）            | 同上（一齊當 head）            |
| 尾   | 253  | 決策大腦權重                  | 機率 **×0.35**、幅度 **×0.45** |

白話：**多用時間試「邊啲指標、點調」**；大腦唔好亂郁得太勁。  
插班生：只重抽頭（參數 + mask），保留尾。

#### 大腦結構（神經網絡開時）

```
17 個輸入 → 10 個隱藏 → 5 個隱藏 → 3 個輸出（買／持／賣）
```

17 維大致：快慢均線、威廉、ROC、RSI、MACD、保力加、波動、成交量、N 日新高、持倉、同 RSI／威廉距離門檻……  
**關咗嘅指標家族會填 0**；持倉狀態永遠開。

#### 9 個可開關指標家族

移動平均線、威廉、ROC、RSI、MACD、保力加、波動率、成交量、N 日新高。

#### 分數權重（fitness）

- 只用時間 **前 80%** 做訓練分數；**後 20% 測試唔入收生**
- 每筆買賣成本 **0.1%**（`0.001`）
- 起始虛擬本金 **10,000**

```
總分 ≈ 全段分數×0.82 + 半段穩健分數×0.18
        − 大腦 L2 罰（權重平均平方 × 1.0）
        − 指標太多罰（見下）
```

**一段分數（全段同每一半都用呢套）：**

| 項目                      | 權重      | 白話                                           |
| ------------------------- | --------- | ---------------------------------------------- |
| 年化回報 CAGR             | × **130** | 長期賺得快好                                   |
| 累積回報                  | × **55**  | 成段賺幾多（畫面「訓練回報」）                 |
| Sharpe                    | × **10**  | 回報穩唔穩                                     |
| 最大回撤                  | × **−22** | 跌得深要扣                                     |
| 超額 vs 死買唔賣          | × **22**  | 贏大市有獎；輸大市扣                           |
| 平均倉位暴露              | × **8**   | 鼓勵有持倉，唔好永遠空手                       |
| 近乎零回報（\|r\|&lt;3%） | **−14**   | 躺平偷懶                                       |
| 牛市低倉又遠輸大市        | **−16**   | 牛市驚到唔敢買（暴露&lt;30% 且回報遠低於大市） |

**半段穩健：** 訓練再切兩半，  
`0.35 × 較差半 + 0.65 × 兩半平均`  
（一個半段差少少唔會完全打死）。

**指標太多罰（sparsity）：**

| 規則           | 數值      |
| -------------- | --------- |
| 前幾個開着免費 | **5** 個  |
| 之後每個再開   | **−0.65** |
| 全部指標都關   | **−80**   |

#### 規則模式（關神經網絡時）

唔用 253 個權重；改為：

- 開着嘅 SMA／MACD／RSI／威廉 **多數贊成** → 買
- RSI 或威廉過熱（如果有開）→ 賣
- 兩個出場都關 → 買入條件唔再成立就離場

#### 畫面額外顯示

- 訓練／測試回報、死買唔賣對照
- 指標開／關 chips
- **Ablation：** 逐個關指標睇分數 Δ（正 = 有貢獻；≤0 = 可能多餘）
- 可匯出 Pine Script

## 股票頁：點解有「訓練」同「測試」？

想像你溫書：

- **前 80% 日子** = 溫習卷（訓練）  
  電腦用呢段時間計分數、揀邊個學生生仔。
- **後 20% 日子** = 真正考試（測試）  
  **唔參與收生**。只係俾你睇：溫習好嘅人，考生卷係咪一樣叻。

如果訓練好靚、測試好醜，好多時係 **背熟舊卷、唔係真本事**（過度擬合）。  
呢個正正係股票 demo 想教嘅一課。

## 常見問題

### 點解股票訓練好慢？

因為每一個學生，都要喺好多年日線上面「假裝交易」一次。  
學生愈多、歷史愈長，愈慢。

想快：

- 播放速度拉去 **5**
- 族群細少少
- 試下關神經網絡（規則模式）

### 點解訓練回報靚、測試又仆？

好正常。  
等於背熟溫習卷，生卷唔識。  
記住：**測試段唔入訓練**，就係要揭穿呢種假叻。

### 搵到「必勝參數」未？

**未，亦唔應該咁諗。**  
呢頁示範嘅係「點樣用進化去搜尋」，唔係保證未來賺錢。

### 匯出 Pine Script 係咩？

把而家冠軍策略，寫成 TradingView 用得嘅腳本格式。  
一樣：**只供學習，唔係實盤建議。**

## 一句收尾

EvoLab = **一班隨機策略考完試、好嘅生仔、仔會突變、一代代變叻少少**。

三個遊戲三個考場；計分權重唔同，但進化流程同一套。  
玩得開心，學到「進化點運作」就夠——**唔好當提款密碼。**
